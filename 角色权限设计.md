## 一、角色定义

### 1.1 Web 后台角色

| 角色 | 枚举值 | 说明 |
|------|--------|------|
| 超级管理员 | `super_admin` | 平台级管理，管理所有门店和系统配置 |
| 门店管理员 | `merchant_admin` | 单门店管理，管理本门店的业务数据 |

### 1.2 小程序端角色

| 角色 | 枚举值 | 说明 |
|------|--------|------|
| 普通顾客 | `customer` | 普通用户，只能预约和查看自己的预约 |
| 待审核店长 | `pending_owner` | 已申请入驻，等待审核 |
| 店长 | `owner` | 门店拥有者，管理门店所有业务 |
| 店员 | `staff` | 门店员工，有限的业务操作权限 |

## 三、当前存在的问题

### 3.1 严重问题

#### 1. 交易/记账路由缺少权限控制
**文件**: `packages/server/src/routes/transaction.ts`
```typescript
// 当前：任何登录用户都能创建交易
router.post('/transactions', authMiddleware, transactionController.createTransaction)

// 应该：
router.post('/transactions', authMiddleware, requireOwner, transactionController.createTransaction)
```

#### 2. 服务管理路由缺少权限控制
**文件**: `packages/server/src/routes/service.ts`
```typescript
// 当前：任何登录用户都能增删改服务
router.post('/', authMiddleware, serviceController.createService)

// 应该：
router.post('/', authMiddleware, requireOwner, serviceController.createService)
```

#### 3. 预约路由权限不明确
**文件**: `packages/server/src/routes/appointment.ts`
- 创建/更新/取消预约没有区分是顾客操作还是店长操作
- 顾客应该只能操作自己的预约
- 店长应该能操作本门店所有预约

### 3.2 中等问题

#### 4. 店员角色权限不清晰
- 后端 `requireOwner` 中间件允许 `staff` 访问
- 但前端没有为 `staff` 定义菜单
- 需要明确 `staff` 能做什么、不能做什么

#### 5. 门店归属校验缺失
- 多个路由获取门店详情时未校验用户是否有权访问该门店
- 例如：`/api/merchants/:id` 可以查询任意门店

#### 6. 路由守卫角色检查不完整
**文件**: `packages/admin/src/router/index.ts`
```typescript
// 当前：只检查 admin 和 super_admin
if (user.role !== 'admin' && user.role !== 'super_admin')

// 应该：
if (!['super_admin', 'merchant_admin'].includes(user.role))
```

## 四、改进方案

### 4.1 新增权限中间件

在 `packages/server/src/middleware/auth.ts` 中添加：

```typescript
/**
 * 要求门店归属权限
 * 检查用户是否有权访问特定门店的数据
 */
export function requireMerchantAccess(ctx: Context, next: Next) {
  const userRole = ctx.state.user?.role
  const userMerchantId = ctx.state.user?.merchant_id
  const targetMerchantId = ctx.params.merchant_id || ctx.params.id || ctx.query.merchant_id || ctx.request.body?.merchant_id
  
  // 超管可以访问所有门店
  if (userRole === 'super_admin') {
    return next()
  }
  
  // 其他角色只能访问自己归属的门店
  if (!targetMerchantId || userMerchantId !== targetMerchantId) {
    ctx.status = 403
    ctx.body = { code: 403, message: '无权访问该门店数据', data: null }
    return
  }
  return next()
}

/**
 * 要求顾客权限（只能操作自己的数据）
 */
export function requireCustomer(ctx: Context, next: Next) {
  const role = ctx.state.user?.role
  const allowedRoles = ['customer', 'owner', 'merchant_admin', 'super_admin', 'staff']
  
  if (!allowedRoles.includes(role)) {
    ctx.status = 403
    ctx.body = { code: 403, message: '无权操作', data: null }
    return
  }
  return next()
}
```

### 4.2 修复路由权限

#### transaction.ts
```typescript
// 交易记录 - 需要门店管理权限
router.post('/transactions', authMiddleware, requireOwner, transactionController.createTransaction)
router.get('/transactions', authMiddleware, requireOwner, transactionController.getTransactions)
```

#### service.ts
```typescript
// 服务查询 - 仅需登录（顾客也需要查看服务列表）
router.get('/', authMiddleware, serviceController.getServices)

// 服务增删改 - 需要门店管理权限
router.post('/', authMiddleware, requireOwner, serviceController.createService)
router.put('/:id', authMiddleware, requireOwner, serviceController.updateService)
router.delete('/:id', authMiddleware, requireOwner, serviceController.deleteService)
```

#### appointment.ts
```typescript
// 查询可用时间段 - 公开
router.get('/slots/available', appointmentController.getAvailableSlots)

// 顾客创建预约 - 仅需登录（后端验证逻辑处理）
router.post('/appointments', authMiddleware, appointmentController.createAppointment)

// 查看、更新、取消预约 - 需要登录，后端根据身份判断权限
router.get('/appointments', authMiddleware, appointmentController.getAppointments)
router.get('/appointments/:id', authMiddleware, appointmentController.getAppointmentDetail)
router.put('/appointments/:id', authMiddleware, appointmentController.updateAppointment)
router.delete('/appointments/:id', authMiddleware, appointmentController.cancelAppointment)

// 确认预约 - 需要店长权限
router.post('/appointments/:id/confirm', authMiddleware, requireOwner, appointmentController.confirmAppointment)
```

#### merchant.ts
```typescript
// 获取门店详情 - 需要登录 + 门店归属校验
router.get('/:id', authMiddleware, requireMerchantAccess, merchantController.getMerchant)

// 更新门店 - 需要店长权限 + 门店归属校验
router.put('/:id', authMiddleware, requireMerchantAccess, requireOwner, merchantController.updateMerchant)

// 打烊管理 - 需要店长权限 + 门店归属校验
router.post('/:id/closed-periods', authMiddleware, requireMerchantAccess, requireOwner, merchantController.createClosedPeriod)
router.get('/:id/closed-periods', authMiddleware, requireMerchantAccess, merchantController.getClosedPeriods)
```

### 4.4 明确店员权限

如果需要支持 `staff` 角色，建议：

1. **定义店员权限范围**：
   - 查看预约
   - 修改预约状态（确认到店、开始服务、完成服务）
   - 查看顾客信息
   - 创建交易记录
   - 不能：修改门店设置、管理服务项目

2. **添加店员专属中间件**：
```typescript
export function requireStaff(ctx: Context, next: Next) {
  const role = ctx.state.user?.role
  const allowedRoles = ['owner', 'merchant_admin', 'super_admin', 'staff']
  
  if (!allowedRoles.includes(role)) {
    ctx.status = 403
    ctx.body = { code: 403, message: '需要员工权限', data: null }
    return
  }
  return next()
}
```
